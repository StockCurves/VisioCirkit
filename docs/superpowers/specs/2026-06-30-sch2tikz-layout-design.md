# sch2tikz 佈局與美感改善設計規格 (sch2tikz Layout & Aesthetics Design Spec)

本文件定義改善 `sch2tikz` 轉換電路圖之佈局、對齊與美感的規範，包括元件邊界框預算、直角避讓 Clearances、以及如實繪製刻意斜線的標準。

## 1. 目標 (Objective)
- 提升 `sch2tikz` 的自動排版美感，防止元件擠壓重疊或標籤文字遮擋導線。
- 消除因坐標微小偏差導致的非預期斜線，使標準電路呈直角整齊對齊。
- 對於電路中刻意設計的斜向線路（如 Latch 的回饋斜線、整流橋等），必須忠實保留與繪製。

---

## 2. 設計細節 (Design Details)

### A. 元件與文字邊界框規範 (Bounding Box Specs)
為避免重疊與擁擠，AI 規劃坐標時，各主要單元需分配以下「虛擬邊界框 (Bounding Box)」空間預算（單位：cm）：

| 元件/區域類型 (Type) | 寬度預算 (Width) | 高度預算 (Height) | 說明 (Description) |
| :--- | :--- | :--- | :--- |
| **比較器 / 運放** (`op amp` / `comparator`) | `2.4 cm` | `1.2 cm` | 包含端子接線之最小保護包絡線 |
| **多輸入邏輯閘** (`nand`, `and`, `or` port) | `1.6 cm` | `1.0 cm` | |
| **單輸入邏輯閘** (`not` port) | `1.4 cm` | `0.6 cm` | |
| **雙端元件** (`R`, `C`, `L`, `opening switch`) | `1.0 cm` | `0.6 cm` | 本體一般為 `0.8cm` |
| **文字標籤** (Text Label Node) | `1.5 cm` | `0.5 cm` | 必須預設使用 `\small` 並設定明確 anchor 與對齊 |
| **波形示意圖** (Timing Waveform Sketch) | `4.5 cm` | `2.5 cm` | 必須置於主要電路軌跡外側，不得干涉電路元件 |

---

### B. 走線與避讓規則 (Routing & Clearance Rules)
1. **非預期斜線消除**：
   - 凡原圖中呈水平或垂直之走線，輸出之 TikZ 坐標必須嚴格對齊（例如：相連的 R 輸出坐標與 OP 輸入腳的 y 坐標必須完全相等，例如皆為 `y = 6.64`）。
   - 走線與並行的元件邊界之間，必須保留至少 `0.6cm` 的安全距離 (Clearance)。
2. **刻意斜線如實呈現**：
   - 若使用者提供的 schematic 中**含有刻意設計之斜線**（例如：NAND Latch 交叉回饋線、H 橋、電橋、橋式整流器），**AI 必須如實使用絕對坐標繪製出斜線**，不得將其強制拉直。
3. **交叉連接點**：
   - 導線相交連接時，必須使用 `\node[circ] at (x, y) {};` 明確標記連線點，未標記點代表互不相干的交叉跨越。

---

### C. 區塊間距與對稱性 (Block & Symmetry)
1. **區塊隔離**：
   - 不同的電路模組（如：輸入級、控制邏輯級、輸出級）之間，水平/垂直方向上必須預留至少 `1.5cm` 以上的 Clear Gap，防止密集壓迫感。
2. **對稱分支對齊**：
   - 對稱的電路結構（例如 Latch 的對稱上下分支、比較器的雙參考電壓輸入分支）必須以對稱軸為準，遵循對等且完全對稱的相對坐標偏移。

---

## 3. 測試與驗證計劃 (Verification Plan)
- **單元測試**：在 `SKILL.md` 更新後，上傳帶有斜線（如 Latch）與無斜線（如基本放大器）的 schematic 進行測試。
- **視覺檢驗**：產生的向量 SVG 應確認：
  - 1. 無非預期的微小斜線。
  - 2. 斜向回饋線如實保留。
  - 3. 標籤與元件本體、走線之間沒有任何重疊。
